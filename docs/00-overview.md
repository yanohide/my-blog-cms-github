# はじめに：全体像と読み進め方

このドキュメントは、**開発経験がない方**でもブログサイトの構築・運用ができるように書かれた手順書です。**4 つのステップ**を 1 つずつ進めていきましょう。

---

## このプロジェクトで使うもの

```
┌───────────┐   ┌───────────┐   ┌───────────┐
│  Sanity    │   │  GitHub   │   │  Vercel   │
│ (記事の倉庫)│──▶│(コードの保管庫)│──▶│(サイトの公開場所)│
└───────────┘   └───────────┘   └───────────┘

┌───────────┐   ┌───────────┐
│  Obsidian  │   │  Cursor   │
│ (ネタ帳)   │──▶│(記事を書く+操作する)│
└───────────┘   └───────────┘
```

| ツール/サービス | 役割 |
|---------------|------|
| **Sanity** | ブログ記事のデータを保管する倉庫 |
| **GitHub** | コードの変更履歴を管理するバックアップ |
| **Vercel** | サイトをインターネットに公開する場所 |
| **Obsidian** | ネタ集め・調査メモ・アウトラインを整理するノートアプリ |
| **Cursor** | 記事の執筆、コード編集、AI によるコマンド操作を行うエディタ |

---

## 4 ステップの全体像

**1 ステップずつ進めてください。** 各ステップの最後に「ここまでの確認」があるので、それがクリアできたら次に進みましょう。

```
Step 1  ブログサイトをローカルで動かす
        Sanity + Next.js のセットアップ → npm run dev で動作確認
        ┃
Step 2  サンプル記事を作って GitHub と連携する
        記事を作成 → Sanity にアップロード → GitHub にプッシュ
        ┃
Step 3  ブログサイトを Web で公開する
        Vercel にデプロイ → git push するだけで自動反映されることを確認
        ┃
Step 4  日常の記事運用をはじめる
        Obsidian でネタ集め → Cursor で記事執筆 → Sanity にアップロード
```

## 事前に必要なもの

手順を始める前に、以下のものを準備してください。

### 必須

- **Google アカウント**（Gmail アドレス）
  - Sanity や Vercel のアカウント登録に使います
- **ウェブブラウザ**
  - Google Chrome を推奨します

### インストールするもの

| ツール | 用途 | ダウンロード |
|--------|------|------------|
| **Node.js** | プログラムを動かすために必要 | [https://nodejs.org](https://nodejs.org)（LTS 版） |
| **Cursor** | コード・記事の編集、AI によるコマンド操作 | [https://cursor.sh](https://cursor.sh) |
| **Obsidian** | ネタ帳、調査メモ、アウトライン作成 | [https://obsidian.md](https://obsidian.md) |

> **Node.js** のバージョンは **20.9 以上** が必要です。現在の LTS 版（Node.js 24）なら問題ありません。
>
> **LTS 版とは？** Long Term Support（長期サポート）の略。安定して長く使えるバージョンです。
